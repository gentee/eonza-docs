# Хранилище

Хранилище даёт возможность безопасно хранить и использовать в скриптах секретную информацию, такую как пароли и токены. Перед началом использования хранилища, его необходимо создать. Для этого укажите мастер-пароль, который будет использоваться для расшифровки и шифрования хранилища по алгоритму **AES-256**.

%html.pro-storage-create%

Хранилище хранится на диске только в зашифрованном виде. Если вы забудете мастер-пароль, то данные невозможно будет восстановить. Следует заметить, что после запуска программы хранилище находится в зашифрованном виде. Укажите мастер-пароль на верхней панели или на закладке хранилища, чтобы Eonza расшифровала данные. В противном случае, строки вида **#@storagename#** не будут заменятся на соответствующие значения из хранилища при выполнении скриптов.

%html.pro-storage-master%

Если мастер-пароль указан правильно, то хранилище будет расшифровано и вы можете добавить туда пароли, токены и другую важную информацию. Колонка с секретной инфоромацией в таблице не отображается.

%html.pro-storage%

**Отключить**  
Отключить хранилище. После нажатия на эту кнопку работа с хранилищем будет отключена и в дальнейшем необходимо будет снова вводить мастер-пароль.

**Изменить пароль**  
Изменить мастер-пароль. Для изменения мастер-пароля нужно знать текущий пароль.

## Добавление и редактирование

%html.pro-storage-item%

**Имя**  
Имя пароля или токена, которое можно будет указывать в скриптах для подстановки соответствующего значения. Для этого используйте запись **#@thisname#**. Например, вы добавили пароль для доступа к базе данных с именем *mypassdb*. В этом случае, указывайте в скриптах в качестве пароля *#@mypassdb#*.

**Описание**  
Описание или комментарий для этого элемента хранилища.

**Значение**  
Пароль или токен, которые необходимо сохранить в хранилище. При редактировании записи, текущее значение не отображается, поэтому оставляйте это поле пустым или указывайте новое значение.
