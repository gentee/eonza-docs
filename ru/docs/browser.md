# Браузер

**v1.20.0**  
Существует [расширение для Chrome](chrome-extension.html), с помощью которого можно запускать скрипты Eonza. Здесь вы можете указать скрипты, которые будут доступны для запуска в Chrome расширении. Если вы используёте Pro версию и имеется несколько пользователей, то они тоже могут использовать расширение в браузере, но будут видеть только те скрипты,
к которым у них имеется доступ.

%html.browser%

## Список скриптов

Укажите список сайтов и скриптов, которые будут доступны для запуска из браузера.

### URLs

В этом поле следует указать список адресов сайтов. Вы можете указывать по одному сайту на строке или разделять их пробелами.
Если это поле пустое, то соответствующие скрипты будут показываться для всех сайтов в браузере. Если адрес сайта начинается с *http*, то сравнивается начало адресов. Например, вы указали *https://mysite.com/*, то в этом случае указанные скрипты будут доступны в браузере на страницах *https://mysite.com/mypage.html, https://mysite.com/docs*, но будут отсутствовать
на страницах *http://mysite.com/, https://www.mysite.com*. Если адрес не начинается с *http*, то
скрипты будут доступны для всех сайтов, которые содержат указанную подстроку. Например, вы указали *bank*. В этом случае,
скрипты в расширении будут доступны на сайтах *mybank.com, banki.ru, mysite.com/banks.html*.

## Скрипты

Укажите через запятую один или несколько скриптов, которые можно будет запустить из расширения браузера на соответствующих сайтах. По умолчанию, при вызове скрипта в константе *#.data#* передаётся JSON строка cо следующими полями.

* **url** - текущий URL в браузере;
* **title** - заголовок текущей страницы в браузере;
* **html** - исходники текущей HTML страницы. Содержимое страницы передаётся только, если отмечен чекбокс *Получить содержимое HTML*.

%html.browser-edit%

### Получить содержимое HTML

Отметьте этот чекбокс, если вы хотите при запуске скрипта получать содержимое HTML страницы. В этом случае, вам не нужно будет дополнительно запрашивать HTML страницу для парсинга.
